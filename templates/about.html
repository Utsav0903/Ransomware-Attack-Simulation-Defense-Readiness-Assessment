What is RADSAS?
RADSAS is a full-stack, interactive ransomware simulation and defense readiness platform built for cybersecurity education. It simulates a complete ransomware lifecycle — from initial reconnaissance through file encryption to forensic recovery — inside a fully safe, isolated sandbox environment. The system features a cinematic 3D SOC globe, a live system terminal, attack timeline tracking, and automated forensic report generation.

⚔ ATTACK LIFECYCLE (6 PHASES)
00
IDLE
System in standby mode. No active threat detected. All files safe and accessible.
01
RECON
Attacker fingerprints the system, scans directories, and identifies high-value targets.
02
INFILTRATE
Payload delivered. Memory hooks injected. Persistence mechanism established in registry.
03
LATERAL MOVEMENT
Malware spreads across connected drives, network shares, and accessible file systems.
04
ENCRYPT
AES-128 cipher applied to all victim files. Original data overwritten. Ransom note dropped.
05
COMPLETE
All files encrypted. System compromised. Decryption key held for simulated ransom payment.

🔌 SYSTEM ARCHITECTURE
The system follows a client-server model.

The Flask backend manages all state via JSON files and exposes a REST API (/status, /attack, /decrypt) polled by the frontend every 1 second.

Encryption and decryption run in background daemon threads so the dashboard remains responsive during operations.

State is fully reset on every dashboard reload for a clean simulation session.
⚠ Ethical Use Notice
RADSAS is intended strictly for controlled, educational environments. The encryption engine operates only on files within the designated victim_files/ sandbox directory. No real systems, networks, or external files are affected. Using techniques demonstrated here against real systems without authorization is illegal and unethical. This tool must only be used for learning and research purposes.
